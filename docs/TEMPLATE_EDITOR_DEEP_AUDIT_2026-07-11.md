# 模板编辑器交互级审查（2026-07-11）

## 审查方法

本轮不以模板列表缩略图为结论。实际登录竞品账号，创建一份简历并进入编辑器，执行模块显隐、换色、字体/字号、更换模板和一页适配操作；对单栏、左右栏和表格模板读取实际渲染 DOM、尺寸与样式变化。

本审查用于提炼通用产品能力，不复制第三方源码、素材或逐像素视觉成品。

## 实际观察

### 全局编辑器

- 编辑器顶部提供下载、发邮箱、分享；左侧为排版能力，右侧为内容模块。
- 模板更换后保留原简历内容、已选主题色、字体和字号。
- 模块支持显隐；开启空模块后立即出现新增条目按钮和排序控制。
- 字体：微软雅黑、宋体、黑体、楷体、苹方、Arial。
- 字号：12–16。
- 语言：简体中文、繁体中文、英文、西班牙文、法文、日文、韩文。
- 主题色：16 个固定色板加自定义色。固定色为：
  `#333333 #545969 #284967 #4e7282 #3978a3 #1575bf #0e88ad #5695c3 #609ef3 #a08f75 #c19f67 #ed7d31 #67a886 #76ba31 #f36c6c #a05fca`。
- 间距包含三个独立维度：模块上下间距、正文行间距、页面边距。
- 内容达到两页时显示页数和“自动调整为1页”；压缩不隐藏模块，而是调整排版参数。
- 标题设置包含“简历标题”和可选 Slogan。
- 封面、自荐信是独立页面能力，不混入正文模块。

### 单栏模板 1045

- 内容区宽 820px；模块默认上间距 25px。
- 主题颜色同时作用于模块标题文字、分隔线和装饰元素。
- 字号从 13 调为 15 后，条目主信息、正文说明同步放大，模块标题保持更高层级。
- 开启“项目经验”后，画布生成空模块，同时右侧出现“新增一条项目经验”。

### 左右栏模板 1015

- 总画布宽 820px、最小高度 1160px。
- 左栏固定 235px，背景使用主题色；右栏为 583px 内容区。
- 左栏承载照片、个人信息、技能说明和技能进度；右栏承载姓名、意向、教育与工作经历。
- 技能不是单一标签：同一模块内同时存在富文本说明与 95%/65% 进度条。
- 主题色改变后，左栏背景、姓名、意向、模块图标、模块边框、经历时间与时间轴节点同步改变。

### 表格模板 1001

- 基本信息使用真实 HTML 表格，而不是普通模块加边框。
- 表格共 6 行；首行 5 个单元格，其余信息行按 4–5 个单元格组合。
- 照片使用 `rowspan=3`；求职意向等字段使用 `colspan=2`。
- 标签单元格使用主题色的低透明度背景，标签文字使用主题色。
- 正文模块仍保持独立排版，基本信息表只解决高密度结构化字段。

### 非对称分栏模板 1034

- 左栏承载姓名、求职意向、个人信息、教育、技能与证书；右栏承载照片、自我评价和工作经历。
- 自我评价位于照片下方形成导语区，不使用普通模块标题；工作经历才进入右栏主叙事。
- 左右栏不是简单的“侧栏 + 全部正文”：模块按语义重新分配，且照片与摘要组成右栏顶部视觉锚点。
- 技能进度同时显示百分比和“精通/良好”等文字等级。

## 实施原则

1. 先补通用排版引擎能力，再扩充模板数量。
2. 每套模板必须有独立信息结构和打印验证，不以换色冒充新模板。
3. 一页适配不得删除、隐藏或改写用户内容；达到可读性下限仍溢出时必须明确提示。
4. 表格类模板应使用语义表格或等价的稳定网格，照片跨行和字段合并需要成为布局规则。

## 本轮结构复刻落地（2026-07-11）

- “正式表格 · 规范信息版”已改为语义化 `<table>`：基本信息 5 行，照片 `rowspan=3`，邮箱与求职意向使用跨列单元格；求职意向不再作为重复正文模块渲染。
- 修复模板名称存在多个关键词时的错误匹配：从“命中第一个关键词”改为按命中数量和关键词具体度评分，表格模板列表卡片现在能稳定进入 `qm-table-formal` 布局。
- 左右栏模板按竞品实测比例落地：820px 画布下左栏 235px、右栏 585px，左栏比例 28.66%。
- 技能条目新增 `proficiency` 和 `level` 字段；旧数据缺少字段时继续以名称正常展示，新数据可显示百分比、文字等级和主题色进度条。
- 删除侧栏右侧误用的大圆点时间轴，恢复为竖向色条、模块标题和细分隔线，正文起始线统一。
- 浏览器验收：表格宽 736px、无横向溢出；侧栏 235/585；4 个技能进度条；浏览器无运行时错误。

## 登录与保存链路修复（2026-07-11）

- 修复 `127.0.0.1:5173` 被本地后端 CORS 拒绝并错误表现为 500：本地 loopback 地址现在在相同端口下兼容 `localhost`、`127.0.0.1` 和 `[::1]`，生产域名白名单不会扩大。
- 增加 CORS 策略回归测试，并以 `testuser` 完成浏览器真实登录，成功跳转到 `/resumes`。
- 修复旧简历携带已删除模板 ID 时无法保存的问题：后端验证模板是否仍存在，失效外键降级为 `NULL`，JSON 中的模板名称、布局与样式继续保留。
- 新增简历标题、Slogan 和页面边距，保存后重新打开可完整恢复；56px 边距实测无横向溢出。
- PDF 导出入口实测完成，无错误提示，后端无新增 CORS、数据库或导出异常。
