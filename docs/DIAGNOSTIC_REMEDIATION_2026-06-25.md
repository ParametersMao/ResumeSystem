# 诊断整改台账（2026-06-25）

## 结论

本轮基于外部体检报告完成了可在代码与本地环境闭环的高风险整改。报告中的 5 个 P0 并非 5 个完全独立的代码缺陷，其中 HTTPS、Git 历史密钥和数据库迁移属于部署或架构治理事项，不能仅靠局部代码修改宣告完成。

## 本轮已完成

### Agent 与 RAG 安全

- Agent 与 RAG 业务端点增加 `X-Agent-Secret` 内部鉴权，健康检查保持可探测。
- NestJS 不再把模型 API Key 放入 Agent 请求体，Python Agent 只从自身环境变量读取密钥。
- Agent 与 Qdrant 取消宿主机端口映射，仅在 Docker 内部网络通信。
- RAG 上传增加 10 MB 限制，检索增加最低相似度阈值。

### AI 业务一致性

- `diagnose` 按 `executionEngine` 在直连模型与 Agent 之间正确分流。
- 直连模式新增真实诊断能力，Mock 模式提供结构兼容的诊断结果。
- Mock AI 不再消耗用户真实额度；真实模型调用仍按原逻辑扣减并在失败时返还。
- AI 操作日志只保存任务元数据、内容长度、模型和结果数量，不再保存完整简历正文与生成正文。

### 配置与前端

- 中台补齐 SMTP 主机、端口、账号、授权码、发件人和加密方式配置界面。
- 系统配置接口不再回传 SMTP 密码和模型 API Key，只返回“已配置”标记。
- 空密码保存时保留服务端原密钥，避免中台读取遮罩后误清空。
- C 端增加 HTTP 429 友好提示和重试等待信息。

## 已验证

- 后端构建通过。
- C 端和中台生产构建通过。
- Python Agent 全量语法编译通过。
- Jest：5 个测试套件、69 个测试全部通过。
