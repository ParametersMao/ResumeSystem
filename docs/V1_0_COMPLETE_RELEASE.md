# ResumeSystem v1.0 完整版发布说明

发布日期：2026-07-10

## 发布结论

本版本达到本地完整版本候选（Release Candidate）标准：核心简历闭环、岗位定向、版本管理、文件导入、十套结构化模板、PDF 导出、RAG 知识库、Agent 降级与可观测性均有可运行实现和回归证据。

线上旧版是否启用 RAG 仍由服务器部署参数决定；本地完整栈已启动并通过端到端验证。

## 用户闭环

1. 选择模板或导入 TXT/PDF/DOCX 旧简历。
2. 在导入预览中核对个人信息与模块文本，确认后创建新简历，不覆盖原文件。
3. 编辑个人信息、教育、工作、项目、实习、校园、技能、荣誉、总结等模块。
4. 输入公司、岗位与 JD，查看关键词证据覆盖和缺口所在模块。
5. 运行 AI 诊断/润色/生成；RAG 不可用时返回明确降级结果。
6. 保存岗位版本、查看历史版本并回滚。
7. 预览并导出 PDF。

## 模板体系

当前数据库种子包含 10 套独立信息结构：

- 技术开发 ATS 单栏
- 产品运营成果导向
- 应届校招项目实习优先
- 蓝色右上证件照
- 正式表格规范信息版
- 左栏头像信息版
- 居中标题头像版
- 深蓝横条紧凑版
- 图标时间轴版
- 商务高管深色版

模板共用编辑数据模型，但拥有独立 DOM/CSS 结构、头像策略、模块顺序、可见模块和打印样式。颜色、字体、字号、行距与模块间距可继续由编辑器调整。

设计基准借鉴公开竞品的结构分类与产品能力，不复制第三方模板源码、素材或逐像素视觉成品。

## 导入能力

- 格式：TXT、PDF、DOCX
- 文件上限：10MB
- 安全策略：先解析预览，用户确认后新建；不覆盖现有简历
- 风险提示：扫描 PDF 无文本时提示 OCR；分栏 PDF 提醒核对阅读顺序
- 识别范围：姓名、电话、邮箱及常见中文简历模块标题

## RAG / Agent

- Agent：FastAPI + LangGraph（依赖缺失时有本地流程降级）
- 向量库：Qdrant
- 本地默认嵌入：384 维中文字符 n-gram 哈希，免模型下载、可重复
- 生产推荐嵌入：`BAAI/bge-small-zh-v1.5`
- 来源追踪：每条结果返回 `sourceId=documentId:chunkIndex`、文档名、分类、片段与相似度
- 防护：知识库内容按不可信数据隔离；新增数字若无用户原文证据则产生风险警告
- 指标：索引文档数、分块数、检索数、命中数、失败数、最近检索耗时与错误

## 启动方式

核心栈：

```bash
docker compose up -d mysql backend web admin
```

完整 RAG 栈：

```bash
docker compose --profile rag up -d
```

访问地址：

- 用户端：http://localhost:5173
- 后端健康检查：http://localhost:3000/api/health
- 管理端：http://localhost:3030

## 上生产前必须确认

- 替换所有示例密钥，保持 `.env` 不入库。
- 配置真实模型 API Key；若使用 Agent，设置 `AI_EXECUTION_ENGINE=agent`、`AGENT_EXECUTION_MODE=live`。
- 用 `docker compose -f docker-compose.prod.yml --profile rag up -d --build` 启动生产完整栈。
- 对生产知识文档执行一次索引和检索验收。
- 配置数据库、对象存储与 Qdrant 数据卷备份。
