# 简历模板布局契约 v3

更新日期：2026-07-14
适用范围：模板卡片、模板预览、简历编辑器、打印预览、PDF 导出

## 1. 目标与对标结论

本轮不再把“上传后能看到照片”作为完成标准，而是把照片、身份信息、模块内部条目和分页视为同一套版式系统。

对标全民简历的模板详情页和真实编辑态后，确认成熟模板具备以下共同结构：

1. 照片默认占位，不因内容为空而让头部结构塌陷。
2. 照片与姓名、求职意向、联系方式共用稳定的头部栅格，而不是悬浮盖在文字上。
3. 公司、岗位、日期有明确主次和对齐规则；长文本只在自己的列内换行。
4. 教育、工作、项目等条目作为一个语义整体排版，标题不会孤零零留在上一页。
5. 短内容保持呼吸感，长内容通过列宽、行高、条目间距和自然分页吸收，不靠任意压缩字号。

本项目只吸收布局规律、信息层级和抗异常内容策略，不复制对方品牌、插画、图标或原始模板资产。

## 2. 统一照片契约

统一图像规则：

- `object-fit: cover`
- `object-position: center 20%`，优先保留头顶、眼睛和肩部构图
- 默认纵横比接近 4:5，不使用圆形头像冒充证件照
- 图片加载失败时回退到职业照占位，不让布局坍塌
- 模板卡片、编辑器、打印 HTML 和 PDF 必须使用同一份契约

## 3. 头部与基本信息布局

### 3.1 栅格原则

- 有照片头部使用显式 grid/flex 列：`正文 minmax(0, 1fr) + 照片固定宽度`。
- 姓名、岗位、联系方式容器都必须设置 `min-width: 0`，避免长邮箱把整页撑宽。
- 电话、邮箱、城市等字段使用自适应网格；允许字段自身换行，不允许与照片重叠。
- 邮箱、个人网站等长连续字符串先在 `@ . _ / -` 后插入零宽换行机会，值节点使用 `word-break: normal; overflow-wrap: normal`；禁止在普通英文单词中间强拆。
- 空字段直接不渲染；不保留空标签、空分隔符或固定高度空行。

### 3.2 长短内容策略

- 短内容：信息块保持垂直居中或顶部基线对齐，不以大段空白填充照片高度。
- 长内容：先增加信息区行数，再自然增加头部高度；不得缩小照片或把字号压到不可读。
- 多个联系字段不追求每行数量一致，追求列宽和基线稳定；窄字段可跨列，长邮箱独占较宽列。

## 4. 模块内部条目契约

工作、教育、项目、实习、校园经历统一采用语义三段式：

```text
主标题（公司/学校/项目）    次标题（岗位/专业）    日期
职责、成果或课程内容
```

排版规则：

1. 第一行优先使用 `minmax(0, 1fr) auto`，主标题和岗位可在左侧内部换行，日期保持完整并右对齐。
2. 当可用宽度不足时，条目头切换为两行：第一行主标题，第二行岗位在左、日期在右；不把日期挤成逐字换行。
3. 公司/学校为一级强调，岗位/专业为二级强调，日期为弱化信息；三者不能同粗同黑。
4. 描述段落用真实列表结构；圆点、悬挂缩进和正文左边线保持一致。
5. 条目之间的间距大于条目内部段落间距，形成清晰分组。
6. 空描述、空公司、空日期不产生占位行；部分字段缺失时仍保持合法布局。
7. 连续中文、英文 URL、邮箱和超长技术名词均不得产生水平溢出。

### 4.1 双栏模板的模块路由

- `qm-spotlight-featured`：技能、荣誉、兴趣进入 240px 侧栏；项目、工作、实习、教育、校园、自定义经历进入主栏；求职意向进入 Hero。
- `qm-asymmetric-profile`：技能、荣誉、兴趣进入 40% 辅助栏；教育、工作、实习、项目、校园、自定义经历进入 60% 主栏。
- `qm-student-editorial`：教育摘要使用 `118px 标签 + minmax(0, 1fr) 列表`，多条教育记录必须全部位于列表列，不能成为外层 Grid 的直接自动布局项。
- 长叙事模块不得因为 DOM 顺序或通用过滤逻辑落入窄侧栏；自动化测试通过 `data-section-type` 和模板结构契约验证模块归属。

## 5. 密度与分页契约

- A4 内容区使用稳定物理尺寸；浏览器预览和 PDF 共用打印样式。
- 模块标题与其后第一条内容保持在同页，禁止标题孤行。
- 单个经历条目默认避免跨页；条目过长无法整体容纳一页时，才允许在段落边界自然拆分。
- 非末页不应出现可容纳两行以上正文的无理由大空白。
- 短数据集目标 1 页；长数据集允许自然多页，不把 `scrollHeight > 1 页` 误判为溢出。
- PDF 返回页数必须与实际 PDF 页数一致。
- 照片必须以 PDF 图像对象或等价的真实像素内容嵌入，而不是只存在于浏览器预览。

## 6. 兼容与数据迁移

- 旧版 `enabled:false + placement:hidden + 0×0` 仅在精确命中已知回归签名、且不存在用户可见性覆盖时升级为当前模板默认照片契约。
- 模板默认行为与未来的用户主动覆盖分离：默认值来自模板；用户明确操作才可记录 `visible/hidden`。
- 生产模板只允许按既有 ID 原位更新 JSON，禁止删除后重建。
- 历史上无模板关联或引用已停用模板的简历不在本轮强制改绑。

## 7. 发布门禁

每套模板必须同时通过：

1. 无照片占位、真实横图裁切、真实竖图裁切三种照片状态。
2. 短姓名与超长姓名、普通邮箱与超长邮箱、短岗位与超长岗位。
3. 短数据 1 页和长数据自然多页。
4. 无水平溢出、无照片遮挡、无标题孤行、无日期逐字换行。
5. 模板卡片、详情、编辑器、刷新后状态和 PDF 的照片位置一致。
6. 旧隐藏签名自动恢复；已有显式可见配置保持可见。
7. 11/11 layout key 在 API 往返后不丢失，后台保存模板后不重新隐藏照片。

自动化压力数据必须克隆各模板的原生经历条目，禁止向 section/Grid 根节点临时追加伪造标题或段落。门禁还必须断言：

- 11 套默认数据全部恰好 1 页；统一长数据夹具为 2–4 页，且不得显著偏离中位页数。
- 公司、岗位、日期标记数量精确一致；日期始终单行。
- 压力条目全部位于模板主内容根，侧栏压力条目数为 0。
- 不存在零宽文本、逐字竖排、元素级横向溢出或普通英文单词中段换行。
- Formal 描述仍位于原生 article 内，并保持四向单元格 padding。
- 浏览器长图只验证内容流；最终发布还必须用 `buildCoreResumePrintHtml` 生成真实 PDF，再逐页渲染为 PNG 做视觉复核。

### 7.1 当前自动化证据

- `qa:template-photos`：11/11 布局分别验证默认照片尺寸/位置、短内容单页、长内容自然分页、主栏条目归属、日期单行和横向溢出。
- `qa:template-pdf`：11 套模板各生成 short/long 两份真实 A4 PDF，共 22 份；照片必须保留为同源 `/uploads/resume-photos/user-{id}/...` URL，经 HTTP 解码后以 raster 对象写入 PDF，禁止用 data URL 绕过资源链路。
- `backed-resume qa:pdf-upload`：直接执行生产 `ResumesService.exportPdf`，验证用户归属、Storage 下载、Chromium 解码、导出对象上传以及最终 PDF 图片绘制指令。
- short 数据必须恰好 1 页；统一 long 夹具允许 2–3 页。标题与首条同页、普通条目不得跨页，禁止空白尾页。
