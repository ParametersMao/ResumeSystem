# JD 驱动的中文简历优化系统路线图

更新时间：2026-07-10

## 产品目标

不与成熟平台竞争“模板数量”，而是围绕一个更明确的用户结果构建产品：

> 用户导入或创建一份事实简历，提供目标岗位/JD，系统在不编造经历的前提下完成差距诊断、逐条改写、岗位专属版本保存和 ATS 安全导出。

核心承诺：

- 真实：不主动补造公司、学校、时间、证书和量化结果。
- 定向：每次优化都对应一个目标岗位或一份 JD。
- 可控：AI 只生成建议，用户确认后才写入简历。
- 可追溯：保留修改前快照、岗位版本和 AI 操作记录。
- 可投递：优先保证一页密度、打印稳定性和 ATS 可解析性。

## 北极星流程

```text
创建或导入事实简历
  -> 选择目标岗位并粘贴 JD
  -> 生成岗位匹配诊断
  -> 逐条确认优化建议
  -> 保存为岗位专属版本
  -> 使用 ATS 安全模板导出 PDF
```

## 阶段与验收

### M0：可信度与定位收口

目标：所有公开页面只承诺已经存在的能力，用户能在 30 秒内理解产品定位。

验收：

- 首页不再展示未经真实数据支持的用户数、满意度和模板数量。
- 不再宣称尚未支持的多格式导出。
- 模板列表不展示种子使用量，改为版式特点和适用场景。
- 首页、模板页和编辑器对“岗位定向、事实保留、版本可追溯”的表达一致。

### M1：六套旗舰模板

目标：用少量高质量模板覆盖主要中文招聘场景。

模板族：

1. 技术开发 ATS 单栏。
2. 产品/运营成果导向。
3. 校招应届生。
4. 财务/行政正式版。
5. 销售/市场业绩版。
6. 中高层管理版。

验收：

- 每套模板拥有独立信息架构，而不只是换颜色和标题装饰。
- 使用一份内容完整的真实感示例数据制作卡片和预览。
- 卡片、预览、编辑器、PDF 四处视觉一致。
- A4 单页和双页场景均无截断、遮挡、孤行和异常空白。
- 技术 ATS 模板通过文本复制、阅读顺序和关键词解析检查。

### M2：岗位/JD 工作区

目标：让用户围绕一次真实投递完成诊断和改写。

范围：

- 新增目标岗位和 JD 输入。
- 提取职责、必备技能、加分项和关键词。
- 输出有证据的匹配项、缺口和风险，不生成虚假匹配分。
- 支持从诊断项直接进入对应简历模块修改。
- 保存为“公司 + 岗位”专属版本。

验收：

- 相同简历针对两份不同 JD 能产生明显不同、可解释的建议。
- 每条建议能指出依据来自 JD、用户简历或知识库。
- 建议应用前后自动创建版本快照。
- 用户可以恢复到修改前版本。

### M3：旧简历导入

目标：降低首次使用成本。

范围：PDF、DOCX、TXT 导入；结构化识别个人信息、教育、工作、项目、技能；低置信字段要求用户确认。

验收：

- 文本型中文 PDF/DOCX 的核心字段识别率达到可接受基线。
- 原文始终可查看，解析结果不会静默覆盖。
- 扫描件和解析失败场景提供明确兜底。

### M4：生产级 RAG 与评测

目标：证明 RAG 确实提升建议质量，而不是只增加系统复杂度。

范围：权限隔离、索引一致性、混合检索、重排、Prompt Injection 防护、引用脱敏、超时重试和可观测性。

验收：

- 建立不少于 50 组“简历 + JD + 人工标准”的固定评测集。
- 对比无 RAG、向量检索、混合检索三组结果。
- 记录相关性、事实保持、建议可应用率、延迟和成本。
- 停用、删除、重建索引在数据库和向量库之间保持一致。

### M5：商业化与分发

目标：在核心求职闭环有效之后再建设付费能力。

候选方向：

- B2C：岗位版本数、AI Credit、旗舰模板和导出权益。
- B2B2C：学校就业中心、求职服务机构、企业内推项目的私有知识库和运营后台。

## 当前优先级

1. 完成 M0，消除不真实承诺。
2. 重做三套首发旗舰模板：技术 ATS、校招、产品/运营。
3. 实现 M2 的最小闭环，而不是继续扩充普通模板。
4. 补 M3 导入，降低用户开始成本。
5. 完成 RAG 安全与评测后再部署到生产。

## 产品指标

- 首次完成率：注册后成功保存并导出第一份简历的用户占比。
- 岗位版本创建率：使用 JD 诊断后保存专属版本的用户占比。
- 建议应用率：AI 建议被用户实际应用的比例。
- 事实退回率：用户因 AI 编造或失真而撤销建议的比例。
- 导出成功率：PDF 导出成功次数 / 发起次数。
- 模板切换后保留率：更换模板后继续编辑或导出的比例。
