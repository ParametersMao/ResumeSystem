# SaaS 第一阶段交付说明：账号、存档与免费权益

## 阶段边界

本阶段只实现非支付能力。支付渠道、订单、退款和付费套餐购买放到第二阶段。

## 已完成

- 邮箱验证码注册、邮箱验证码登录、用户名或邮箱密码登录。
- 邮箱验证码找回密码，修改密码后旧刷新令牌立即失效。
- 服务端安全退出，主动使已有刷新令牌失效。
- 邮箱身份表与验证码表，验证码只保存哈希值。
- 验证码 10 分钟有效、60 秒发送冷却、最多 5 次校验。
- SMTP 配置复用中台系统配置；未配置 SMTP 时生产环境拒绝发送。
- C 端简历、版本、照片与 PDF 接口统一使用 JWT 身份。
- 服务端不再信任客户端提交的 `userId`。
- 跨用户读取、修改和删除简历返回 404。
- 后台用户、统计、AI 操作日志、系统日志和模板写操作要求管理员身份。
- 免费版后端强制额度：
  - 最多保存 2 份简历。
  - 每份简历保留最近 5 个历史版本。
  - 每月 10 次 AI 操作。
  - 每月 3 次 PDF 导出。
  - 100 MB 简历数据存储额度展示。
- AI 和 PDF 使用“先预占、失败返还”的额度扣减方式。
- 个人中心展示简历、AI、PDF 和存储用量。
- Docker 初始化脚本包含账号身份、验证码和权益字段。

## 生产环境必须配置

```env
EMAIL_CODE_SECRET=至少32位随机字符串
EMAIL_ALLOW_DEV_CODE=false
```

并在中台“系统配置”中填写 SMTP 主机、端口、账号、密码、发件邮箱和加密方式。

`EMAIL_ALLOW_DEV_CODE=true` 只用于本地验收，禁止用于公网生产环境。

## 已验证

- 后端、C 端和中台生产构建通过。
- 后端 Jest：69/69 通过。
- Docker API 冒烟测试通过：
  - 邮箱注册成功。
  - 新用户默认获得 Free 权益。
  - 前两份简历可创建，第三份返回 403。
  - 用户 A 无法读取用户 B 的简历。
  - 未登录无法访问后台统计接口。
