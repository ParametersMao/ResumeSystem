# v0.2 岗位定向投递工作区

发布日期：2026-07-10

## 版本定位

v0.2 将产品从“通用在线简历编辑器”推进为“围绕一份真实 JD 制作岗位专属简历版本”的最小可用闭环。

核心流程：

```text
选择信息结构 → 编辑事实简历 → 填写公司、岗位和 JD
→ 检查 JD 要求与简历文字证据 → 定位到对应模块修改
→ 调用 AI/Agent 做深度诊断 → 保存公司 · 岗位专属版本
```

## 本版本交付

### 岗位定向工作区

- 支持填写目标公司、目标岗位和最多 8000 字的职位描述。
- 从 JD 中提取产品、运营、增长、数据、协作和技术工具关键词。
- 对照当前简历可见模块，展示关键词是否存在直接文字证据。
- 覆盖率只表示文字证据覆盖，不包装成不可解释的“岗位匹配分”。
- 对缺口生成优先修改清单，并可定位到技能、项目、经历、教育或个人优势模块。
- 定位操作只展开编辑模块，不自动写入或虚构内容。
- 将 JD 与当前简历一并传给现有 AI/Agent 深度诊断接口。
- AI 不可用时仍保留本地证据覆盖分析。
- 将公司、岗位、JD、关键词和诊断时间写入简历文档。
- 支持保存带有“岗位版本：公司 · 岗位”备注的版本快照。

### 首发三套旗舰模板

1. 技术开发 · ATS 单栏投递版。
2. 应届校招 · 项目实习优先版。
3. 产品运营 · 成果导向版。

第三套模板新增：

- 项目优先于工作经历展示。
- 以业务问题、个人行动和指标结果组织内容。
- 自动从描述中提取百分比、数量和排名作为成果信号。
- 能力、教育和求职定位收纳到侧栏，正文聚焦代表成果。
- 网页预览与 PDF 打印样式保持一致。

### 本地开发闭环

- Mock 模式补齐数值型简历 ID、创建、更新、详情和列表接口。
- Mock 模式补齐版本创建、列表与回滚接口。
- Mock 模式补齐岗位深度诊断响应。
- 可以在不启动后端数据库和外部模型的情况下完成前端端到端验收。

## 已知边界

- 当前关键词抽取是可解释的规则与词典方案，不是语义匹配模型。
- 关键词存在只代表简历中出现直接文字证据，不代表候选人一定满足岗位要求。
- 本地 Mock 数据保存在浏览器运行内存中，页面完整刷新后不会持久化；生产环境由数据库持久化。
- 本版本不会自动应用 AI 修改，仍由用户审核后写入。
