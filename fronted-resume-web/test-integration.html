🔍 完全自定义样式功能集成测试

测试步骤

  1. 启动开发服务器：npm run dev
  2. 打开简历编辑器页面
  3. 在浏览器开发者工具中检查以下内容：

1. 检查CSS注入

在开发者工具的 Elements 标签中，检查 <head> 部分是否有：

<style data-resume-custom="true" type="text/css"> /* 自定义CSS内容 */ </style>
注意： 如果看不到这个style标签，说明CSS注入功能没有正常工作。

2. 检查样式应用

在开发者工具的 Elements 标签中，检查简历预览区域是否有：

  • .dynamic-resume-preview 元素有红色边框和渐变背景
  • .resume-content 元素有蓝色边框和圆角
  • 如果使用了测试模板，应该有明显的视觉变化

3. 检查CSS变量

在开发者工具的 Elements 标签中，检查 :root 是否有CSS变量：

:root { --colors-primary: #3498db; --colors-secondary: #2ecc71; --spacing-md: 16px; --spacing-lg: 24px; }

4. 检查响应式样式

在开发者工具的 Elements 标签中，检查是否有媒体查询：

@media (min-width: 768px) { .some-class { font-size: 16px; } }

5. 检查Schema渲染

如果使用了Schema配置，检查是否有：

  • .schema-renderer 类名的元素
  • .schema-grid 或 .schema-flex 类名的元素
  • 自定义的样式是否正确应用

6. 调试信息

在浏览器控制台中运行以下代码来检查功能状态：

// 检查是否有自定义样式元素 const customStyle = document.querySelector('style[data-resume-custom="true"]'); console.log('自定义样式元素:', customStyle); // 检查样式内容 if (customStyle) { console.log('样式内容:', customStyle.textContent); } // 检查CSS变量 const rootStyles = getComputedStyle(document.documentElement); console.log('CSS变量 primary:', rootStyles.getPropertyValue('--colors-primary')); console.log('CSS变量 spacing-lg:', rootStyles.getPropertyValue('--spacing-lg'));

常见问题排查

问题1： 看不到自定义样式元素
解决： 检查模板数据中是否有 customCss 字段，或者检查组件是否正确加载
问题2： 样式没有生效
解决： 检查CSS选择器是否正确，或者是否有样式冲突
问题3： Schema渲染不工作
解决： 检查 SchemaRenderer.vue 是否正确导入到 BaseSection.vue 中

预期结果

✅ 成功标志：
  • 在 <head> 中看到 data-resume-custom="true" 的style标签
  • 简历预览区域有明显的视觉变化（如果使用了测试模板）
  • CSS变量在 :root 中定义
  • 响应式样式通过媒体查询实现
  • Schema渲染的组件正确显示
