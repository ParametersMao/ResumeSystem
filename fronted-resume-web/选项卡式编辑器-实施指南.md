## 🚀 使用方法

### 方法1：在 ResumeEditorNew.vue 中使用（推荐）

1. **导入组件**

```vue
<script setup lang="ts">
import TabModuleEditor from '@/components/editor/TabModuleEditor.vue'
</script>
```

2. **替换现有编辑器**

找到这段代码：
```vue
<ModuleListEditor 
  ref="moduleListRef"
  :resume-data="resumeData"
  @ai="onAiFromSection"
/>
```

替换为：
```vue
<TabModuleEditor 
  :resume-data="resumeData"
  @update:resumeData="handleResumeDataUpdate"
  @change="handleResumeChange"
  @ai-optimize="handleAiOptimize"
/>
```

3. **添加事件处理函数**

```typescript

  resumeData.value = newData
}

  // 触发自动保存
  debouncedSave()
}

// 处理 AI 优化请求

  const { moduleType, data } = payload
  
  // 打开 AI 面板
  aiVisible.value = true

  // 这里可以复用现有的 AI 优化逻辑
  console.log('AI 优化请求:', moduleType, data)
```


## 🎨 特性说明

### 1. 顶部选项卡

- **显示所有模块**：默认显示12个模块的选项卡
- **状态指示**：
  - ✅ 绿色勾：已启用的模块
  - 灰色：未启用的模块
  - 蓝色高亮：当前激活的模块
- **点击切换**：点击选项卡切换到对应模块

### 2. 模块管理

- **启用/禁用**：底部有"启用/禁用此模块"按钮
- **基本信息**：始终启用，无法禁用
- **默认状态**：根据模板配置决定哪些模块默认启用

### 3. 表单功能

每个模块包含：
- **必填字段标记**：带 * 的字段为必填
- **上移/下移**：调整条目顺序（多条目模块）
- **删除按钮**：删除当前条目
- **新增按钮**：添加新条目

### 4. AI 优化

- **模块级优化**：底部的"AI 优化"按钮
- **条目级优化**：某些模块（如工作经验）每条都有独立的优化按钮
- **触发事件**：`@ai-optimize` 事件，包含模块类型和数据

### 5. 实时保存

- 每次输入都会触发 `@change` 事件
- 建议使用防抖处理，避免频繁保存


3. 在 `TabModuleEditor.vue` 中导入并使用：

```vue

import NewModuleEditor from './modules/NewModuleEditor.vue'
</script>

<template>
  <!-- 添加条件渲染 -->
  <NewModuleEditor
    v-if="activeModule === 'newModule'"
    v-model="getSectionData('newModule')"
    @change="emitChange"
  />
</template>
```

### 自定义验证

在各个编辑器组件中添加验证逻辑：

```typescript
function validateForm() {
  if (!localData.value.name) {
    ElMessage.error('姓名不能为空')
    return false
  }
  return true
}

function emitChange() {
  if (validateForm()) {
    emit('update:modelValue', localData.value)
    emit('change', localData.value)
  }
}
```

## ⚠️ 注意事项

### 1. 数据同步

确保 `resumeData` 的变化能正确传递到编辑器：

```typescript
// 使用 watch 监听数据变化
watch(() => props.resumeData, (newData) => {
  // 更新本地数据
}, { deep: true })
```

### 2. 防抖保存

避免频繁保存：

```typescript
import { debounce } from 'lodash-es'

const debouncedSave = debounce(() => {
  // 保存逻辑
}, 800)
```

### 3. 模块顺序

sections 数组的 `order` 字段决定渲染顺序，确保正确设置。

### 4. 富文本编辑

如果需要富文本编辑器，可以集成 Quill 或 TinyMCE：

```bash
npm install @vueup/vue-quill
```

```vue
<template>
  <QuillEditor v-model:content="content" contentType="html" />
</template>
```
