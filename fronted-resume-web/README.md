简历编辑器 Web 端落地方案（fronted-resume-web）

## 1. 背景与目标

本方案参考主流在线简历平台的编辑体验与导出质量，结合现有中台/后端能力，为 `fronted-resume-web/` 提供一份可直接落地的实现蓝图：

- 目标用户：需要快速制作高质量 PDF 简历的 C 端用户。
- 关键能力：模板化呈现、区块化编辑、所见即所得预览、即时保存、稳定导出（PDF/图片）、分享与版本化。
- 对标参考：全民简历编辑器入口、登录与使用路径（微信扫码、账号登录、同意协议）[参考](https://www.qmjianli.com/cv/edit/?resumeId=2494ZLD3D61WCO)。

## 2. 技术栈与工程基座

- 框架：Vue 3 + Vite + TypeScript
- 状态：Pinia（模块化 store），搭配 `@vueuse/core` 提升组合式能力
- UI：UnoCSS/Tailwind（主题可控）+ 少量自研组件（区块卡片、拖拽句柄、预览画布）
- 表单与校验：Zod（模式与运行时校验统一）或 Vuelidate
- 拖拽排序：SortableJS（区块/条目级排序）
- 富文本控件：tiptap（受控精简格式：加粗、斜体、链接、项目符号）
- 路由：Vue Router（登录、模板、编辑器、预览、导出）
- 请求：Axios（拦截器统一鉴权、重试、节流）
- 监控与埋点：Sentry + 自研埋点（UV/PV、导出转化、错误率）
- 打包与部署：Vite 构建，多环境变量（dev/stage/prod），通过 CDN/静态托管（含 `/_headers` 缓存控制）

## 3. 顶层架构与模块划分

- 登录与权限：支持微信扫码/账号登录；统一 Token 鉴权；匿名态仅允许模板浏览与示例编辑。
- 模板与样式：模板库管理（主题色、字体、排版规则），模板切换保持内容结构不丢失。
- 编辑器：
  - 左侧：区块树（新增/隐藏/排序/克隆）
  - 中间：所见即所得预览（受控渲染器、对齐线、吸附）
  - 右侧：样式面板（主题色、字体、字号、行距、页边距、分栏）
- 保存与版本：输入即保存（去抖 800ms），差量提交，本地缓存与离线恢复，乐观并发。
- 导出与分享：客户端快速导出（html2canvas + jsPDF），服务端一致性导出（Puppeteer/Playwright）预留；分享只读链接。

## 4. 目录结构建议

```
fronted-resume-web/
  src/
    api/                # Axios 封装与业务 API
    components/         # 通用组件（区块卡片、表单控件、拖拽句柄、预览画布）
    editor/             # 编辑器域（section 组件、渲染器、样式面板）
      blocks/           # 各类区块实现（工作/项目/教育/技能/荣誉/自我评价）
      renderers/        # 渲染器映射，每类 section 对应一个 renderer
      panels/           # 样式与模板切换面板
    pages/              # 路由页面（登录/模板库/编辑器/预览/导出）
    store/              # Pinia store（用户、模板、简历、导出任务）
    styles/             # 全局样式、主题、打印样式
    utils/              # 通用工具（节流去抖、格式化、分页与缓存）
  index.html
  vite.config.ts
```

## 5. 关键交互与 UX 细节

- 区块级操作：新增/克隆/删除/上下移动/隐藏显示；支持从模板库拖入常用模块 Preset。
- 条目级操作：新增/删除/拖拽排序；标签/星级/图标等可选增强；行内快捷格式（加粗、链接、项目符号）。
- 预览：准像素对齐，吸附与等距分布；选中态高亮对应 DOM；切换主题即时生效。
- 校验：必要字段提示（职位、公司、时间段等），日期区间合法性校验；辅助输入（时间选择、技能等级选择器）。
- 无障碍/键盘：Tab/Enter 快速创建条目，Esc 取消编辑；语义化标签提升可读性与 ATS 兼容。

## 6. 数据与状态（高阶概念）

- 统一简历 Schema：内容（sections/items）与样式（style/layout）解耦，支持模板切换。
- 版本与并发：`updatedAt` + `contentHash` 乐观锁；本地快照（IndexedDB/LocalStorage）回滚。
- 即时保存：输入 → 去抖 → 差量比对 → 失败重试，离线队列同步。

## 7. API 约定与分页规范

- 基础响应：`{ code, message, data }`
- 分页列表：`{ list, total, page, limit }` 统一返回结构，避免直接返回数组，便于前端表格与翻页缓存复用（与既有约定一致）。
- 鉴权：HTTP Header `Authorization: Bearer <token>`，401 统一跳转登录。

## 8. 导出与打印

- 客户端：`html2canvas + jsPDF` 快速导出（DPI/字体/分页优化）。
- 服务端：`Puppeteer/Playwright` 渲染强化一致性（字体嵌入、分页控制、孤行处理）。
- 打印 CSS：分页符、页边距、字体回退、图像压缩；预检页预览。

## 9. 性能与质量

- 性能：图片懒加载、区块虚拟滚动、CSS 变量热切换；避免过度富文本。
- 稳定性：Sentry 错误收集；接口重试与降级（只读预览/导出离线包）。
- 测试：组件单测 + 关键流程 E2E（登录 → 编辑 → 保存 → 导出）。

## 10. 部署与监控

- 环境：`VITE_API_BASE` 分环境配置；静态站点托管（CDN 缓存、HTTP/2）。
- 监控：前端性能指标（LCP/FID/CLS）、导出成功率、错误 TopN；灰度发布与回滚。
