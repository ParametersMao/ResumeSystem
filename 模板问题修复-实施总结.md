
### 数据流

```
模板文件 (.json)
    │
    ▼
[解析] → 模板对象 (Object)
    │
    ▼
[诊断] → DiagnosticResult
    │      ├─ isValid: boolean
    │      ├─ format: 'new'|'old'|'unknown'
    │      ├─ errors: DiagnosticError[]
    │      ├─ warnings: DiagnosticWarning[]
    │      ├─ fixes: DiagnosticFix[]
    │      └─ canAutoFix: boolean
    │
    ▼
[修复] → 修复后的模板
    │
    ▼
[验证] → 重新诊断确认
    │
    ▼
[导出] → 新的模板文件
```

## 🎯 核心代码示例

### 诊断模板
```typescript
import { diagnoseTemplate } from '@/utils/templateDiagnostics'

const result = diagnoseTemplate(templateData)

// result 包含:
{
  isValid: true,                    // 是否有效
  format: 'old',                    // 格式类型
  errors: [],                       // 错误列表
  warnings: [                       // 警告列表
    {
      type: 'UNKNOWN_MODULE_TYPE',
      message: '未知的模块类型: personal',
      path: 'layout[0].type',
      suggestion: '将被作为自定义模块处理'
    }
  ],
  fixes: [                          // 可用修复
    {
      type: 'ADD_DEFAULT_CONFIG',
      description: '为模块 basic 添加默认配置',
      autoApplicable: true
    }
  ],
  canAutoFix: true                  // 可自动修复
}
```

### 自动修复
```typescript
import { autoFixTemplate } from '@/utils/templateDiagnostics'

if (result.canAutoFix) {
  const fixed = autoFixTemplate(templateData, result)
  // fixed 是修复后的模板对象
}
```

### 批量诊断
```typescript
import { batchDiagnose } from '@/utils/templateDiagnostics'

const templates = [
  { id: '1', data: template1 },
  { id: '2', data: template2 },
  // ...
]

const results = batchDiagnose(templates)
// results 是 Map<string, DiagnosticResult>
```
