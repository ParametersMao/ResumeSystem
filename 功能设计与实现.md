第 5 章 详细设计与实现5.1 用户鉴权与安全访问模块用户鉴权是系统的安全底座，通过构建“Bcrypt+JWT”的无状态防御体系，保障用户资产安全 。5.1.1 用户注册与合法性校验（1）功能描述：新用户通过邮箱、用户名及强密码创建系统账户 。系统需执行输入正则校验与数据库去重检测 。（2）实现方案：前端基于 Element Plus 实施表单前置校验。后端 NestJS 接收到 POST 请求后，利用 TypeORM 检索 Users 表的唯一性索引字段 。5.1.2 JWT 无状态鉴权机制（1）功能描述：用户登录成功后获取加密 Token，作为后续私有接口访问的唯一凭证 。（2）实现方案：系统集成 @nestjs/jwt 模块，将 userId 与 role 封装进 Payload 并签发。后端通过全局拦截器 AuthGuard 校验请求头中的 Authorization 字段，确保非法请求无法触达业务层 。5.1.3 密码加盐哈希存储（1）功能描述：严禁明文存储，保障即便数据库泄露，攻击者也无法还原密码 。（2）实现方案：调用 bcrypt 算法执行 10 轮加盐处理，登录时利用 compare 方法进行哈希对比 。5.2 个人资料与头像管理模块负责维护用户的结构化档案，包括个性化配置与多媒体资产管理 。5.2.1 用户基础档案维护（1）功能描述：支持用户修改姓名、联系方式、社交主页等非核心敏感数据 。（2）实现方案：采用 PUT 语义接口，后端通过 JWT 提取 userId，定位数据库行记录并执行 Patch 更新。5.2.2 异步头像上传与 OSS 托管（1）功能描述：支持高分辨率头像上传，并在预览区实时展示 。（2）实现方案：前端将文件序列化为 Base64 或 Multipart 流。后端对接阿里云 OSS，上传成功后返回具备 CDN 加速能力的持久化 URL，并将其写入用户表 avatar 字段 。5.3 简历生命周期管理模块简历副本的 CRUD 是系统的基本盘，通过副本隔离机制支持用户的“多赛道”求职 。5.3.1 多副本列表与分页检索（1）功能描述：用户可查阅名下所有简历实例，支持按更新时间降序排列 。（2）实现方案：后端构建带有 where: { userId } 条件的 findAndCount 查询。通过建立复合索引优化分页性能，确保在大规模副本场景下响应延迟低于 200ms。5.3.2 简历初始化与模板绑定（1）功能描述：基于选定模板创建新简历实例 。（2）实现方案：POST 接口接收 templateId，后端初始化 content 字段为标准的 JSON 数据骨架，确保编辑器加载时具备基本的模块结构 。5.4 响应式简历编辑器引擎这是系统的核心交互区，实现了“所见即所得”的原子化编辑体验 。5.4.1 原子化组件动态渲染（1）功能描述：左侧编辑表单根据简历 JSON 结构动态生成交互控件 。（2）实现方案：利用 Vue 3 的动态组件特性，递归遍历简历树。每个模块（如工作经历）均为独立组件，通过 Props 传递数据，实现逻辑的高度聚合与解耦 。5.4.2 布局动态重排逻辑（1）功能描述：支持通过拖拽或点击排序按钮改变模块先后顺序 。（2）实现方案：前端操作即时修改 Pinia 状态库中的数组索引。右侧预览区感应到数据变动后，通过虚拟 DOM 的高效 Diff 算法触发重新渲染 。5.5 高保真预览渲染引擎确保 Web 端预览效果与 PDF 导出效果的高度一致 。5.5.1 响应式数据驱动同步（1）功能描述：左侧表单输入时，右侧 A4 区域实时同步内容 。（2）实现方案：基于 Vue 3 的单向数据流。引入防抖（Debounce）函数，在用户停止输入 300ms 后触发更新，平衡了渲染流畅度与 CPU 负载 。5.5.2 预览区自适应缩放算法（1）功能描述：在不同尺寸屏幕下完美展示 A4 比例 。（2）实现方案：设计基于 transform: scale() 的动态计算公式。引擎实时检测预览容器宽度，自动计算缩放系数，确保布局不因窗口尺寸变化而崩坏。5.6 AI 智能内容辅助模块利用 LLM 的语义理解能力为简历内容赋能，从工具向服务转型 。5.6.1 提示词工程（Prompt Engineering）构建（1）功能描述：为 AI 提供精准的行业上下文约束，提升生成质量 。（2）实现方案：后端建立 Prompt 模板库。根据用户选择的模块类型（如项目经验），自动注入“量化成果、专业术语、职业动词”等 Prompt 强约束 。5.6.2 AI 异步调用与流式反馈（1）功能描述：处理 AI 调用的长耗时问题，提供良好的用户感知 。（2）实现方案：接口采用流式输出或异步回调模式。前端展示骨架屏加载动画，待结果返回后以卡片形式呈现三种维度的优化建议供用户勾选 。5.7 AI 智能内容生成模块基于岗位关键词快速填充简历骨架 。5.7.1 基于岗位的模块化生成（1）功能描述：输入岗位名称，自动生成专业技能、项目亮点等内容 。（2）实现方案：调用大模型 Completion 接口。系统预设行业知识库，确保生成内容具备高度的行业垂直度与岗位匹配度 。5.8 模板资源中心模块为用户提供多元化、标准化的视觉排版方案 。5.8.1 行业分类检索与预览（1）功能描述：按互联网、金融等行业标签筛选模板 。（2）实现方案：后端 Templates 表维护分类标签。前端通过响应式卡片流展示缩略图，并提供 A4 规格的高清预览视图 。5.9 高保真 PDF 导出模块利用服务端渲染技术，产出具备出版级质量的 PDF 文档 。5.9.1 Puppeteer 无头浏览器渲染流程（1）功能描述：将动态 Web 页面序列化为 PDF 。（2）实现方案：后端启动 Puppeteer 进程，注入用户 JSON 数据至指定 HTML 模板。通过配置 printBackground: true 及 300DPI 参数，保障 PDF 的色彩还原度与文字可搜索性 。5.9.2 云端存储与时效性链接分发（1）功能描述：文件托管至云端，提供安全下载 。（2）实现方案：集成阿里云 OSS。文件上传后生成具备 24 小时有效期的安全下载签名，降低了应用服务器的 I/O 负担 。5.10 简历版本控制与快照模块为简历编辑过程提供“悔恨药”，保障数据资产的绝对安全 。5.10.1 编辑进度自动快照（1）功能描述：每次保存自动创建历史快照 。（2）实现方案：在 Resumes 主表更新前，将当前 content 字段内容全量复制到 ResumeVersions 表 。5.10.2 历史版本回滚与对比（1）功能描述：支持查看历史记录并一键恢复 。（2）实现方案：拉取版本列表，用户选定版本后，将其 JSON 内容覆盖回主表 content 字段，实现数据的秒级回滚。5.11 后端系统治理与审计模块确保管理员对系统运行状态的绝对掌控 。5.11.1 全流程安全审计日志（1）功能描述：记录所有敏感操作记录 。（2）实现方案：设计 NestJS 全局日志中间件。自动采集请求 IP、操作类型、参数及执行耗时，并持久化至 SystemLogs 表供管理员追溯。5.12 管理员：用户与生态治理模块维护平台准入门槛与模板生态 。5.12.1 用户生命周期管控（1）功能描述：账号禁用、激活及重置 。（2）实现方案：通过管理员后台管理界面。针对违规账号，后端在 Redis 缓存中设置黑名单标志位，拦截其所有 JWT 鉴权请求。5.12.2 模板准入审核系统（1）功能描述：对新增模板进行排版合规性审核 。（2）实现方案：管理员执行模拟渲染。若模板在 Puppeteer 环境下出现偏移，则返回修改意见；通过后执行上架，更新前端可见状态。5.13 管理员：资源监控与参数配置模块保障系统运行成本在预算阈值内 。5.13.1 AI 调用成本监控看板（1）功能描述：按用户、时间维度统计 Token 消耗 。（2）实现方案：基于 AIOperations 表生成聚合报表 。管理员可实时查看各模型（DeepSeek/讯飞）的开销明细，实现精细化成本核算 。5.13.2 系统阈值与运行参数调节（1）功能描述：动态调整 AI 调用上限等参数 。（2）实现方案：管理员通过配置界面修改参数。数据存入数据库后同步更新 Redis 缓存，实现系统配置的“秒级下发”而无需重启服务。
