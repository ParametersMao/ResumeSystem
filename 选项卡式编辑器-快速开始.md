## 🚀 5分钟集成

### 步骤 1：在 ResumeEditorNew.vue 中集成

1. **打开文件**：`fronted-resume-web/src/views/ResumeEditorNew.vue`

2. **添加导入**（在 script 部分顶部）：

```typescript
import TabModuleEditor from '@/components/editor/TabModuleEditor.vue'
```

3. **找到并替换**：

查找：
```vue
<ModuleListEditor 
  ref="moduleListRef"
  :resume-data="resumeData"
  @ai="onAiFromSection"
  :highlighted-section-id="highlightedSectionId"
  @highlight="handleEditorHighlight"
  @clear-highlight="clearEditorHighlight"
  @select="handleEditorSelect"
/>
```

替换为：
```vue
<TabModuleEditor 
  :resume-data="resumeData"
  :template-config="templateData"
  @update:resumeData="resumeData = $event"
  @change="debouncedSave"
  @ai-optimize="handleAiOptimize"
/>
```

4. **添加 AI 优化处理函数**（如果还没有）：

```typescript
function handleAiOptimize(payload: any) {
  const { moduleType, data } = payload
  
  // 显示 AI 面板
  aiVisible.value = true
  
  // 设置 AI 优化的内容
  currentSectionForAi.value = {
    type: moduleType,
    data: data
  }
  
  console.log('AI 优化请求:', moduleType, data)
}
```

5. **保存文件**，重启开发服务器：

```bash
npm run dev
```

## 📊 数据流向

```
用户填写表单
    ↓
模块编辑器发出 @change 事件
    ↓
TabModuleEditor 更新 resumeData
    ↓
触发 @update:resumeData 事件
    ↓
ResumeEditorNew 接收更新
    ↓
触发自动保存（debouncedSave）
    ↓
更新到数据库
```

## 💡 提示

### 如果遇到问题：

1. **模块不显示**
   - 检查 `resumeData.sections` 是否正确初始化
   - 查看浏览器控制台是否有错误

2. **选项卡无法切换**
   - 确认 `activeModule` 状态正常
   - 检查点击事件是否绑定

3. **数据不更新**
   - 确认 `@update:resumeData` 事件正确处理
   - 检查数据是否正确传递

4. **样式不对**
   - 检查 Element Plus 是否正确引入
   - 确认 common-styles.scss 已正确导入
