# 其他模块处理方式说明

## 模板上传要求

为适配最新的模板上传规范，所有模块需要遵循以下规则：

- **字段固定，样式可配**：字段结构由前端统一定义（`personalFields.ts`、`sectionTypes.ts`），模板只能控制这些字段的显示、顺序、标签和样式，不能新增字段。
- **布局由模板选择**：通过 `config.layout` 指定布局类型（如 `timeline`、`simple`、`tags`、`flexible` 等），并可以进一步通过 `config` 的各类样式字段精细控制。
- **config 优先级最高**：`layout[i].config.*` 会在渲染时覆盖全局样式；模板可通过该对象修改容器、标题、列表项等全部样式。
- **数据只读**：模板读取 `profile` 和 `sections` 的数据进行渲染，不应尝试改变数据结构。

## 当前实现方式

### 1. **字段定义**（前端固定）

所有模块的字段都在 `fronted-resume-web/src/config/sectionTypes.ts` 中定义：

```typescript
export const SECTION_TYPES: Record<string, SectionConfig> = {
  experience: {
    itemType: "list",
    allowMultiple: true,
    fields: [
      { name: "company", label: "公司名称", type: "text", required: true },
      { name: "role", label: "职位", type: "text", required: true },
      {
        name: "duration",
        label: "时间区间",
        type: "dateRange",
        required: true,
      },
      {
        name: "desc",
        label: "工作描述",
        type: "textarea",
        required: false,
        richText: true,
      },
      { name: "icon", label: "图标", type: "select", required: false },
    ],
  },
  education: {
    itemType: "list",
    allowMultiple: true,
    fields: [
      { name: "school", label: "学校名称", type: "text", required: true },
      { name: "degree", label: "专业学历", type: "text", required: true },
      {
        name: "duration",
        label: "时间区间",
        type: "dateRange",
        required: true,
      },
    ],
  },
  skills: {
    itemType: "array",
    allowMultiple: false,
    fields: [
      { name: "skill", label: "技能名称", type: "text", required: true },
    ],
  },
  // ... 其他模块
};
```

**特点**：

- ✅ 字段结构固定（前端定义）
- ✅ 保证数据一致性
- ✅ 编辑器和渲染器都使用相同的字段定义

### 2. **布局组件**（多种布局可选）

每个模块都有多个布局组件，通过 `config.layout` 选择：

#### 工作经验 (experience)

- `timeline`：时间轴布局 (`TimelineLayout.vue`)
- `simple`：简单列表布局 (`SimpleLayout.vue`)

#### 教育背景 (education)

- `timeline`：时间轴布局 (`TimelineLayout.vue`)
- `simple`：简单列表布局 (`SimpleLayout.vue`)

#### 技能 (skills)

- `tags`：标签布局 (`TagsLayout.vue`)
- `progress`：进度条布局 (`ProgressLayout.vue`)
- `simple`：简单列表布局 (`SimpleLayout.vue`)

### 3. **样式控制**（完全由模板配置）

每个布局组件都通过 `config` 和 `styles` 接收样式配置：

```typescript
// 示例：TimelineLayout.vue
const companyStyle = computed(() => ({
  fontSize: "16px",
  fontWeight: "600",
  color: props.styles?.colors?.text || "#333",
  marginBottom: "5px",
  ...props.config?.companyStyle, // 模板可以覆盖
}));

const positionStyle = computed(() => ({
  fontSize: "14px",
  color: props.styles?.colors?.text || "#333",
  ...props.config?.positionStyle, // 模板可以覆盖
}));
```

## 对比：基本信息模块 vs 其他模块

| 特性               | 基本信息模块                                                          | 其他模块                                   |
| ------------------ | --------------------------------------------------------------------- | ------------------------------------------ |
| **字段定义**       | ✅ 固定（`personalFields.ts`）                                        | ✅ 固定（`sectionTypes.ts`）               |
| **样式控制**       | ✅ 由模板配置                                                         | ✅ 由模板配置                              |
| **布局方式**       | ⚠️ 之前：固定布局（center/table/card）<br>✅ 现在：新增 flexible 布局 | ✅ 多种布局可选（timeline/simple/tags 等） |
| **字段显示控制**   | ✅ 新增：可控制字段显示/隐藏/顺序                                     | ❌ 目前：所有字段都显示                    |
| **字段标签自定义** | ✅ 新增：可自定义字段标签                                             | ❌ 目前：使用固定标签                      |

## 总结

### 相同点

1. ✅ **字段都是固定的**：所有模块的字段都在前端定义
2. ✅ **样式由模板控制**：通过 `config` 可以完全控制样式
3. ✅ **支持多种布局**：每个模块都有多个布局选项

### 不同点

1. **基本信息模块**：

   - 之前只有固定的几种布局（center/table/card）
   - 现在新增了 `flexible` 布局，可以控制字段显示/隐藏/顺序/标签
   - 字段更复杂（包含联系方式、头像、自我概述等）

2. **其他模块**：
   - 布局选择更丰富（timeline/simple/tags/progress 等）
   - 字段相对简单（每个模块的字段数量较少）
   - 目前所有字段都会显示（没有字段显示控制）

## 建议

如果要让其他模块也支持字段显示控制（类似基本信息模块的 flexible 布局），可以：

1. **为其他模块也创建 flexible 布局组件**
2. **或者增强现有布局组件**，添加字段显示控制功能

这样可以让所有模块都支持：

- 控制字段显示/隐藏
- 控制字段显示顺序
- 自定义字段标签
- 完全自定义样式
