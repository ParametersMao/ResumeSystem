## 🎯 立即使用

### 方法一：可视化诊断工具（最简单）

#### 步骤 1: 启动项目
```bash
cd fronted-resume-web
npm run dev
```

#### 步骤 2: 打开浏览器
访问: `http://localhost:5173`（或您配置的端口）

#### 步骤 3: 查看开发工具
在页面右下角会看到两个浮动面板：
- **右下角**: 渲染引擎开发工具
- **右下角偏左**: 🔍 模板诊断工具 ⬅️ **就是这个！**

#### 步骤 4: 使用诊断工具

**单个模板诊断**：
1. 点击"模板诊断工具" → "展开"
2. 选择"单个模板"标签
3. 点击"选择模板文件"或直接粘贴JSON
4. 点击"开始诊断"
5. 查看诊断结果
6. 如果可以自动修复，点击"应用自动修复"
7. 下载修复后的模板

**批量诊断所有模板**：
1. 切换到"批量诊断"标签
2. 点击"开始批量诊断"
3. 系统会自动检查所有模板
4. 查看问题模板列表
5. 点击"查看详情"了解具体问题
6. 导出诊断报告

---

### 方法二：编程方式

如果您需要在代码中使用：

```typescript
import { diagnoseTemplate, autoFixTemplate } from '@/utils/templateDiagnostics'

// 诊断模板
const result = diagnoseTemplate(templateData)

console.log('模板格式:', result.format)  // 'old' | 'new' | 'unknown'
console.log('是否有效:', result.isValid)
console.log('错误数量:', result.errors.length)
console.log('可自动修复:', result.canAutoFix)

// 如果可以修复
if (result.canAutoFix) {
  const fixedTemplate = autoFixTemplate(templateData, result)
  // 使用修复后的模板
}
```

## 📋 已解决的问题

### ✅ 个人信息模块不显示
**原因**: 旧模板使用 `type: "personal"`
**解决**: 自动映射到 `type: "basic"`

### ✅ 模块顺序混乱
**原因**: 缺少 `order` 字段
**解决**: 自动添加默认顺序

### ✅ 样式不生效
**原因**: 旧格式配置未转换
**解决**: 智能适配器自动转换

### ✅ 模块不显示
**原因**: 模块类型拼写错误
**解决**: 类型标准化和诊断提示

### ✅ JSON解析失败
**原因**: 编码问题或格式错误
**解决**: 增强的解析器和错误提示
