# Cursor IDE 规则配置

本文件夹包含针对Cursor IDE的自定义规则配置，旨在提高开发效率、保持代码一致性和提升代码质量。

## 规则概述

### 1. 代码格式化规则

- 使用2空格缩进
- 使用单引号
- 行宽限制为100字符
- 启用尾随逗号
- 箭头函数参数尽可能省略括号

### 2. 文件夹结构规则

- 遵循NestJS模块化结构
- 每个模块包含controller、service和module文件
- DTO和实体类有明确的命名规范和存放位置

### 3. 命名规范

- 类名使用PascalCase，并根据功能添加特定后缀
- 方法和变量名使用camelCase
- 常量使用UPPER_CASE
- 接口名前缀为I并使用PascalCase

### 4. 导入排序规则

- 外部库优先
- 内部模块其次
- 相对路径导入最后

### 5. 代码检查规则

- 禁止未使用的变量
- 禁止隐式any类型
- 限制函数复杂度和参数数量

### 6. 注释规则

- 要求为函数和类添加文档注释
- 标准化TODO格式

### 7. SQL查询规则

- 优先使用QueryBuilder
- 记录生成的SQL查询
- 使用有意义的表别名

## 使用方法

Cursor IDE将自动识别并应用这些规则。你可以通过以下方式使用这些规则：

1. **代码格式化**：使用Cursor IDE的格式化功能时，将应用定义的格式化规则
2. **智能提示**：创建新文件或编写代码时，IDE会根据命名规范提供建议
3. **代码检查**：编写代码时，IDE会根据定义的规则进行实时检查

## 自定义规则

如需修改或添加规则，请编辑`.cursor/rules.json`文件。
