### 1.4 简历导出模块

- **导出PDF实现**：前端将用户编辑好的简历内容渲染为完整HTML，通过接口POST到后端。后端使用puppeteer将HTML渲染为PDF文件，再通过ali-oss SDK上传到阿里云OSS，最后将PDF下载链接返回给前端。这样保证了前后端渲染一致性，导出速度快，且后端实现简单。

- **接口设计**：

  - 请求参数：{ html: '<html>...</html>' }
  - 响应：{ code: 200, message: '导出成功', data: { url: 'https://oss.aliyun.com/xxx/xxx.pdf' } }

### 1.5 多模板简历业务流程说明

- **模板上传与管理（中台/后台）**：后台管理员通过中台系统上传用canvas或其他工具制作的简历模板（HTML片段或结构化数据），后端将模板信息（名称、描述、内容等）存入数据库templates表。

- **C端获取模板并渲染**：C端通过接口获取所有可用模板，前端渲染模板列表供用户选择。

- **用户选择模板并编辑简历内容**：用户在前端选择模板，基于模板编辑自己的简历内容（如填写个人信息、经历等），编辑内容可通过接口保存到数据库（如resume表）。

- **用户导出/下载简历PDF**：用户点击"下载简历"，前端将当前编辑好的内容和所选模板渲染成完整HTML，通过/api/resumes/export接口POST给后端。后端用puppeteer生成PDF并上传到阿里云OSS，返回下载链接给前端，前端直接下载或展示PDF。

- **数据库交互总结**：模板数据由后台上传存储，C端通过接口获取；用户简历内容编辑后保存到数据库，导出PDF时前端渲染HTML，后端仅负责PDF生成和上传，保证渲染一致性和高效性。

### 2.1 性能优化&#xA;

- **数据库查询优化**：对 MySQL 数据库的常用查询语句添加索引，如在`users`表的`username`字段、`resumes`表的`user_id`和`template_id`字段创建索引，加速数据检索。使用 TypeORM 的查询构建器（QueryBuilder）替代部分复杂的原生 SQL 查询，提高代码可读性和查询性能。

- **缓存策略升级**：除 Redis 缓存简历编辑数据外，对不常变化的数据（如模板列表）使用 NestJS 的缓存模块（`@nestjs/cache-manager`）进行内存缓存，设置合理的缓存过期时间，减少数据库查询频率。

- **异步处理**：在`puppeteer`生成 PDF、文件上传等耗时操作中，充分利用 Node.js 的异步特性，使用`async/await`或 Promise，避免阻塞事件循环，提高服务器并发处理能力。
